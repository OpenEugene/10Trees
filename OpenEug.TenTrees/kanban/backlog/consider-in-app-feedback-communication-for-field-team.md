# Consider in-app feedback/communication for field team

## Context

Per the Feb 3, 2026 meeting, Mark offered to build an in-app communication tool so the South Africa team (Quinton, TryGive) can provide feedback on the platform without needing to coordinate across time zones on WhatsApp or scheduled calls.

Mark described it as *"a little user forum kind of thing"* (~9:55) that would allow async feedback between the SA field team and the development team.

## What's Missing

No feature file covers in-app messaging, feedback, or a forum. Currently all communication happens via WhatsApp, email, or scheduled video calls — all of which are difficult given the time zone difference and intermittent connectivity.

## Proposed Feature

This is exploratory — only pursue if the team decides it's worth it. Possible scope:

- [ ] Simple threaded discussion/feedback board within the app
- [ ] Users can post feedback, questions, or bug reports
- [ ] Posts are visible to all staff/admin users
- [ ] Async — no real-time chat requirement
- [ ] Low-bandwidth friendly (text only, no media uploads)
- [ ] Could potentially leverage Oqtane's built-in forum module if available

## Priority

Low — this was a suggestion from Mark, not a hard requirement. The SA team testing via Quinton/TryGive is the immediate driver. May be superseded by just using email or the Oqtane notification system.
